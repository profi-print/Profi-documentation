# ProfitPrint — Система документооборота

Полноценное веб-приложение (SPA) для замены Excel-файлов в типографии.  
Автоматизирует создание, учёт и печать заказов на производство, накладных, актов сверки.  
Данные синхронизируются между веб-интерфейсом, облачной MongoDB и Telegram-ботом.

## Функциональные возможности

- **Дашборд** – статистика, финансовая сводка, последние документы, быстрый старт.
- **Клиенты** – справочник с поиском, создание/редактирование/удаление (в корзину), детализация.
- **Продукты** – номенклатура с артикулом, наименованием, ценой, количеством, стоимостью (авторасчёт), произведено, заказано.
- **Заказы на производство** – создание, редактирование, удаление; автоматическая подстановка данных из продукта; таблица позиций с итогами.
- **Накладные** – создание, в том числе копированием из заказа; автоподстановка количества из продукта; итог скрыт при <2 позициях.
- **Акты сверки** – полностью ручной ввод (как в Excel); добавление строк, автовычисление конечного остатка, итоговая строка с суммами по столбцам.
- **Резервные копии (корзина)** – все удалённые документы сохраняются; просмотр полной информации, восстановление, безвозвратное удаление, экспорт.
- **PDF** – генерация фирменных документов (красный логотип, синие заголовки, числа без "UZS", прочерки вместо нулей).
- **Telegram-бот** – интерактивное меню, получение PDF и детализации документов напрямую из облачной базы.
- **Синхронизация** – между веб-приложением, сервером Node.js и MongoDB Atlas; кнопка «Синхронизировать» на дашборде.
- **Экспорт/Импорт данных** в JSON.

## Архитектура данных

Данные хранятся в трёх слоях:
1. **LocalStorage** браузера (быстрый доступ, офлайн-режим).
2. **MongoDB** (центральное облачное хранилище).
3. **JSON-файлы** (экспорт/импорт, резервные копии).

### Основные коллекции
- **clients** – id, name, contact, phone, address
- **products** – id, sku, name, price, unit, quantity, cost, produced, ordered
- **orders** – id, number, date, completionDate, clientId, items[]
- **invoices** – id, number, date, clientId, items[]
- **payments** – id, clientId, date, amount, description
- **reconciliations** – id, number, date, clientId, periodFrom, periodTo, items[]
- **trash** – массив удалённых объектов с типом, самим объектом и датой удаления

## Веб-приложение (фронтенд)

### Дашборд
- Карточки-счётчики (заказы, накладные, акты, клиенты, продукты)
- Финансовая сводка (реализация, оплаты, задолженность)
- Последние документы с кнопками: **📄 PDF**, **🗑️ Удалить**, **📋 Детали**
- Кнопка **🔄 Синхронизировать** для загрузки данных с сервера
- Быстрый старт (5 шагов) и справка

### Клиенты и Продукты
- Таблицы с поиском, создание/редактирование/удаление (с перемещением в корзину)
- При клике на строку — детальная информация в модальном окне

### Заказы
- Форма с выбором клиента, датами, таблицей позиций
- При выборе продукта автоматически заполняются: артикул, произведено, заказано, цена
- Стоимость пересчитывается автоматически

### Накладные
- Возможность создания из заказа (копирование позиций)
- Количество автоматически подставляется из карточки продукта
- Итого скрывается, если позиций меньше двух

### Акты сверки
- **Полностью ручной ввод** (как в Excel)
- Кнопка **+ Добавить строку** создаёт редактируемые поля
- **Конечный остаток** = Начальный остаток + Реализация − Оплата (автоматически)
- **Итоговая строка** (голубой фон) суммирует все столбцы

### Корзина (Резервные копии)
- Просмотр всех удалённых элементов
- Детализация (полный JSON), восстановление, полное удаление
- Экспорт отдельного элемента или всей корзины

## Серверная часть (бэкенд)

Сервер на Node.js/Express обеспечивает REST API для синхронизации данных с фронтендом и Telegram-ботом.

### API-эндпоинты
| Метод   | Путь                     | Описание                  |
|---------|--------------------------|---------------------------|
| GET     | /api/data                | Получить все данные       |
| POST    | /api/data                | Импортировать все данные  |
| GET     | /api/:collection         | Получить коллекцию        |
| POST    | /api/:collection         | Создать запись            |
| PUT     | /api/:collection/:id     | Обновить запись           |
| DELETE  | /api/:collection/:id     | Удалить запись            |

Коллекции: `clients`, `products`, `orders`, `invoices`, `payments`, `reconciliations`
